# Expense-Tracker-Application

I built this expense tracker to practice working with dynamic UI, localStorage, and CRUD-style logic in vanilla JavaScript. The app lets users add, update, and delete expenses while automatically recalculating totals, average spending, and the most used category. I also wanted the project to feel more polished than a basic practice app, so I designed it with a dark themed interface, category icons, summary cards, and an empty state screen.

This project pushed me more than I expected. One of the things I struggled with most was handling dynamically rendered elements. I ran into problems with event listeners, especially when screens were being replaced with new HTML. At first I thought the logic had to run in a very straight, sequential way, but while building this app I started understanding event delegation, screen re-rendering, and how browser interactions are really driven by user events. I also had to debug issues with form behavior, editing existing expenses, restoring data from localStorage, and keeping track of unique ids for each expense.

In the end, this project helped me get more comfortable with application flow in JavaScript, not just styling or simple DOM changes. It was one of the projects where I had to stop, rethink how the UI and logic should work together, and rebuild parts of it more than once. That made it a frustrating project at times, but also one of the most useful ones I’ve built so far.

## Features

- Add new expenses
- Edit existing expenses
- Delete expenses
- Track total expenses
- Show average spending
- Show top category
- Save data with localStorage
- Restore saved expenses on reload
- Category-based icons
- Empty state screen
- Dark themed UI

## Built With

- HTML
